# 실시간 ROM 교육용 피드백 앱 - Browser-only v3

이 버전은 Streamlit 서버에서 MediaPipe/OpenCV를 실행하지 않고, 사용자의 브라우저 안에서 카메라와 MediaPipe Web을 실행합니다.

## v2 개선 내용

- 스마트폰에서 음성이 나오지 않는 문제를 줄이기 위해 **음성 활성화/테스트** 버튼 추가
- 측정 중 카메라는 유지한 채 분석만 멈추는 **측정 중단** 버튼 추가
- **측정 재개**, **측정 새로 시작**, **카메라 정지** 버튼을 사이드바와 영상 아래에 모두 배치
- 측정 새로 시작 버튼이 측정 중에도 작동하도록 상태 초기화 로직 개선
- 스마트폰에서 음성 안내가 차단될 때 화면 안내 메시지 표시

## 배포 파일

```text
app.py
requirements.txt
packages.txt
README.md
.streamlit/config.toml
```

## requirements.txt

```text
streamlit>=1.38
```

## packages.txt

비어 있어야 합니다.

## 실행

```bash
pip install -r requirements.txt
streamlit run app.py
```

## 스마트폰 사용 안내

- 카카오톡/메신저 내부 브라우저보다 Chrome, Samsung Internet, Safari 같은 일반 브라우저 권장
- HTTPS 주소로 접속
- 카메라 권한 허용
- 음성이 나오지 않으면 먼저 **음성 활성화/테스트** 버튼을 누르고, 스마트폰 무음모드/미디어 볼륨을 확인

## 주의

단일 웹캠 기반 교육/피드백용 프로토타입입니다. 임상 진단 또는 치료 결정에 사용하려면 별도 타당도 검증이 필요합니다.


## v3 개선 내용

- 고정 십자선만 표시하던 방식에서, 신체 기준점에 맞춘 동적 기준선을 추가했습니다.
- 정면 촬영: 코/어깨 중심선, 어깨 수평선, 필요 시 골반 기준선을 표시합니다.
- 측면 촬영: 귀/코-어깨 또는 어깨-골반 수직 정렬선을 표시합니다.
- 음성 안내를 단계별로 명확화했습니다: 준비자세 유지 → 움직임 시작 신호 → 끝범위 후 복귀 신호 → 1회 완료/다음 회 안내.
- 반대 방향 움직임 또는 기준선 불일치 복귀는 기록하지 않고 재정렬 단계로 돌립니다.


## v4 변경 사항

- 음성 안내 문구를 교육용 ROM 측정 절차에 맞게 수정했습니다.
- 움직임 시작 직후 바로 복귀를 안내하지 않고, 선택한 방향으로 끝범위까지 움직인 뒤 끝범위에서 잠시 멈추도록 안내합니다.
- 끝범위에서 약 0.8초 안정적으로 유지되면 “끝범위 움직임이 인식되었습니다. 이제 처음 위치로 천천히 돌아오세요.”라고 안내합니다.
- 끝범위에서 충분히 멈추지 않고 바로 돌아오면 해당 반복을 기록하지 않도록 했습니다.
- 정상 측정 후 다음 회차 안내는 “준비자세로 돌아와 기준선을 다시 맞춰 주세요.”로 통일했습니다.
